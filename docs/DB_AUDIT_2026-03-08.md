## 8) Event System Readiness Notes

Current DB is strong for document-centric policy extraction, but weak for student assignment execution.

- Ready:
  - Rich content corpus in `knowledge_base`/`document_chunks`
  - Metadata and multilingual coverage
  - Embeddings/search vectors already available
- Not ready:
  - No actual `students`, `enrollments`, `assignments` data
  - Identity table quality issues (`users.password_hash`, one `STUDENT` user without `students` record)
  - Some content hygiene issues (empty title/content rows, noisy instructor titles)

## 9) Suggested Next Step for MVP

Use `knowledge_base` as the single source for taxonomy + policy extraction, and treat student assignment as a second phase until SIS transactional tables are populated.
